## Objective
Kiểm thử toàn diện dạng toán mới và đảm bảo không gây hồi quy (regression) cho các dạng toán cũ.

## Requirements
### Functional
- [ ] Kiểm tra luồng Game Over/Victory với dạng toán mới.
- [ ] Kiểm tra trên Mobile (UI 4 thẻ ảnh có bị tràn không).
- [ ] Test tính chính xác của âm thanh trên các trình duyệt khác nhau.

## Implementation Steps
1. [ ] Chạy manual test.
2. [ ] Audit code lần cuối.

## Test Criteria
- [ ] Bé có thể chơi mượt mà từ số 1 đến 10.
- [ ] Các bài toán cộng trừ cũ vẫn hoạt động bình thường.
